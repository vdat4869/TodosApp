## Format for Future Entries

When recording technical debt, use the following format:

### [Debt Item Title]
- **Description:** What the debt is and why it was taken.

- **Impact:** What problems this debt causes (e.g., hard to maintain, slow performance).
- **Remediation Plan:** How and when this should be fixed.
- **Priority:** High / Medium / Low.

## Strategies for Managing Tech Debt

2. **Review regularly:** Review this document during sprint planning.
